## 사용할 기술 스택

**개발 환경 : React Native (Expo Devlopment Build)**

핵심 기술 구현 전략 

- 디자인 생성 및 이미지화
    - UI 렌더링 : Styled-Components + React Native SVG
    - 이미지 변환 : react-native-view-shot

화면상에 컴포넌트 만들고 CSS로 디자인 입히고 , 생성된 임시 파일 경로(url) 받아서 서버에 업로드하거나 저장공간 만들어서 저장

- BLE & Sensor
    - BLE 통신 : react-native-ble-plx
    - 모션 감지 : expo-sensors
    - 설정 플러그인 : @config-plugins/react-native-ble-plx

가속도 센서 사용 해서 충격 오면 뭐 이벤트 나오게 하고 동시에 스캔을 시작해서 RSSI (신호 강도) 아까 테스트 코드 진행하였을 때 두 핸드폰을 가까이 댔을 때 제 핸드폰 기준 -40dBm 정도고 윤성이 핸드폰은 -50dBm 정도 나왔는데 -50dBm 기준으로 필터링 하면 될 거 같슴다

- NFC
    - NFC 제어 : react-native-nfc-manager
    - 모드 : Android (HCE 모드), iOS ( Tag Reader 모드 )
    ## .

이건 솔직히 잘 모르겠는데 찾아본 바에 따르면 Android는 앱 실행하면 약간 삼성페이나 교통카드 같은 상태(?) 로 만들어서 상대방이 리더기로 읽으면 내 UserID가 담긴 NDEF 메시지 송출하는 방식이고 iOS는 다들 아시다시피 백그라운드 송출이 안될거 같고 [NfcTech.Ndef](https://m.blog.naver.com/yh_park02/223498823881) 리더 모드를 활성화 하여 신호를 읽어 들이면 된다고 합니당

- 실시간 데이터 동기화
    - 상태 관리 : TanStack Query(React Query) + Zustand
    - 통신 프로토콜 : WebSocket (Socket.io-client 또는 StompJS)

교환을 성공하면 폭죽 애니메이션이 양쪽 폰에서 동시에 터진다던지 약간 그런 이팩트를 넣고 싶고 데이터는 즉시 갱신되게 하려고 합니당

# 진행 방향 및 목표
- 리스크 관리를 위해 로그인, 회원가입 같은 쉬운 기능은 뒤로 미루고, 이번주 부터 NFC/BLE 통신 테스트부터 수행하여 주요 기술부터 먼저 해결할 생각하고 있슴다
- 이미지 처리 방식이 서버에서 이미지를 만들어서 내려주면 좀 느리다 해갖고 VIew-Shot 기술을 써서 클라이언트가 폰에서 직접 화면을 캡쳐해서 이미지로 바꾸는 방식 있다는데 써보면 어떨까 데이터 절약도 되고 속도도 훨씬 빠르다 하네여
- 블루투스 옆 사람 잡히는거 RSSI(신호강도) 로 필터링 걸어야 할 거 같은데 Gemini 쳐보니까 -50dBm 이상 기기만 골라내고, 동시에 가속도 센서로 ‘쿵’ 하는 충격 시간 비교해서 0.5 초 안에 같이 흔든 사람만 매칭시킨다는데 일단 뭐 테스트코드 로 돌려봤을 때랑 RSSI는 거의 비슷하게 나와서 이렇게 진행해도 괜찮을거 같습니다
