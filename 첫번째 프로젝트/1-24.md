⚡️ 헬스체크 API

Method: GET

URL: /api/v1/health

이건 "서버야, 너 살아있니?" 라고 물어보는 가장 기초적인 API입니다. 이걸로 IntelliJ에서 테스트하는 법을 단계별로 알려드릴게요.

🚀 IntelliJ에서 API 테스트하는 법 (HTTP Client 사용)
IntelliJ에는 **HTTP Client**라는 꿀기능이 내장되어 있습니다. 포스트맨(Postman) 같은 프로그램을 따로 설치할 필요 없이 코드 짜는 곳에서 바로 테스트 가능합니다.

1단계: 백엔드 서버 켜기
먼저 백엔드 서버가 실행 중이어야 합니다. IntelliJ 우측 상단에 **초록색 재생 버튼(▶)**이 눌러져 있고, 아래 콘솔창에 에러 없이 글씨가 떠 있어야 합니다. (보통 포트는 8080을 씁니다.)

2단계: 테스트 파일 만들기
IntelliJ 메뉴에서 File > New > Scratch File을 누르세요.

(단축키: Ctrl + Alt + Shift + Insert)

목록에서 HTTP Request를 찾아서 선택하세요.

그러면 빈 메모장 같은 화면이 나옵니다.

3단계: 코드 작성하기 (복사+붙여넣기)
아까 노션 명세서에서 본 내용을 그대로 적는 겁니다. 아래 코드를 복사해서 그 빈 화면에 붙여넣으세요.

HTTP

### 1. 헬스체크 (서버 살아있는지 확인)
GET http://localhost:8080/api/v1/health
(참고: localhost:8080은 내 컴퓨터의 8080번 포트라는 뜻입니다. 백엔드 설정에 따라 포트 번호가 다를 수 있지만, 보통은 8080입니다.)

4단계: 실행하고 결과 확인하기
작성한 코드 옆을 보면 **초록색 작은 재생 버튼(▶)**이 생겼을 겁니다. 그걸 클릭하세요.

**Run on 'localhost:8080'**을 클릭합니다.

결과 확인: 아래쪽 창에 결과가 뜹니다.

[성공 시 예상 결과] 첫 번째 이미지(image_262db2.png)에 있는 성공 응답 포맷대로 나오면 성공입니다!

JSON

{
  "code": "SU",
  "message": "OK",
  "data": "Health Check OK"  // (이 부분은 개발자가 뭐라 적었냐에 따라 다름)
}
HTTP/1.1 200 OK: 통신 성공했다는 뜻 (초록색 불)

code: "SU": 노션에 적힌 대로 성공 코드가 잘 왔음

💡 요약
명세서(노션)를 본다: "아, GET 방식이고 주소는 /api/v1/health구나."

IntelliJ HTTP 파일에 적는다: GET http://localhost:8080/api/v1/health

누른다: ▶ 버튼 클릭.

본다: 200 OK 뜨면 연결 성공!
