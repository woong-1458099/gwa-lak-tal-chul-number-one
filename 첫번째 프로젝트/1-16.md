1. 테스트 코드로 NFC 태깅 기술과 카카오 SDK api사용해서 연결되는지 확인하기
2. git lab 에 이번주 진행사항, 학습내용 정리하기

AUTH-01	일반 회원가입	사용자는 필수 정보를 입력하여 서비스에 가입한다.	
1. 입력 폼 구현- ID, PW, 이름, 휴대폰, 생일, 성별
2. 주소 찾기 API 연동 (Daum 주소 등)
3. 유효성 검사 (빈칸, 형식 체크)

AUTH-02	중복 가입 방지	휴대폰 번호 또는 이메일이 이미 존재할 경우 가입을 차단한다.	
1. '중복 확인' 버튼 또는 입력 시 실시간 체크
2. 중복 시 에러 메시지 노출("이미 가입된 회원입니다")

AUTH-03	일반 로그인	ID와 비밀번호를 입력하여 서비스에 접속한다.	
1. 로그인 UI 구현
2. 로그인 성공 시 Access/Refresh Token 저장 (SecureStorage)
3. 메인 화면으로 이동

AUTH-04	소셜 로그인	카카오 계정을 연동하여 간편하게 로그인한다. (네이버/구글 후순위)	
1. 카카오 SDK 연동
2. '카카오로 시작하기' 버튼 클릭
3. 인가 코드 수신 후 백엔드 전송

AUTH-05	아이디 찾기	사용자 이름과 휴대폰 번호로 가입된 아이디를 찾는다.	
1. 이름, 휴대폰 번호 입력 폼
2. 결과 모달 출력 ("회원님의 아이디는 user1 입니다")

AUTH-06	비밀번호 재설정	ID, 이름, 전화번호가 일치하면 비밀번호를 변경할 수 있다.	
1. 신원 확인 폼 (ID, 이름, 폰번호)
2. 일치 시 '새 비밀번호 입력' 화면으로 전환
3. 변경 완료 후 로그인 화면 이동

EXCH-01	명함 선택	사용자는 스와이프하여 교환할 내 명함을 선택한다.	
1. Swipe UI 구현
2.  좌우 스와이프시 명함 선택

EXCH-02	명함 교환	NFC 태깅을 통해 상대방을 찾고 명함을 교환한다.	
1. 아래에서 위로 스와이프시 선택한 명함 NFC태깅 기능 트리거
2. NFC HCE/Reader 모드 활성화
3. 교환 성공 모달 출력 ( 누구와 교환했는지도 즉시 보여줌)

EXCH-03	명함 스캔	카메라로 종이 명함을 촬영하여 정보를 인식하고 저장한다.
명함 비율에 맞게 네모난 가이드라인(Overlay) 화면에 띄운다	
1. 카메라 촬영 UI
2. OCR 연동
3. 텍스트 추출 후 폼 입력

EXCH-04	명함 공유	난수 코드를 생성하여 다수에게 내 명함을 공유한다.	
1. '코드 생성' 버튼 클릭 및 UI 표시
2. 상대방 코드 입력 폼 구현

EXCH-05	신규 명함 수신 수락	상대방이 새롭게 생성한 명함을 공유하는 경우, 도착한 신규 명함 공유 요청 알림에서 수락 또는 거절을 선택한다.	
1. 알림/모달에 수락하기/거절하기 버튼 구현
2. 수락 시: “명함첩에 저장됨” 모달 출력
3. 거절 시: 목록에서 요청 삭제

MY-01	명함 생성	새로운 명함을 생성한다. (직접 입력 및 스캔)	
1. 정보 입력 폼 (이름, 소속 프로필 이미지 등)
2. ‘기존 인맥에게 공유하기’ 토글/체크박스 구현
3. 유효성 검사

MY-02	명함 수정	선택한 명함의 정보를 수정한다.	
1. 기존 데이터 불러오기
2. 수정 완료 버튼 액션

MY-03	명함 삭제	더 이상 사용하지 않는 내 명함을 삭제한다.	
1. 삭제 확인 모달 출력
2. 삭제 후 목록 새로고침

MY-04	명함 조회	내 명함의 세부 정보를 조회한다.
1. 명함 상세 페이지 UI
2. 수정 버튼

WALL-01	명함첩 조회	내 명함첩(받은 명함 목록)을 조회한다.	
1. 리스트 UI
2. 검색 필터(이름, 태그) 적용

WALL-02	명함 상세 조회	타인의 명함 상세 정보를 조회한다.	
1. 상세 정보 UI 
2. 전화/문자 버튼 액션 연결

WALL-03	메모 작성	명함 뒷면에 메모(해당 인물에 대한 특이사항)를 작성한다.	
1. 명함 뒤집기 애니메이션 및 메모 UI
2. 메모 입력 모달
3. 작성 후 메모 실시간 반영

WALL-04	명함 삭제	타인의 명함을 휴지통으로 이동시킨다.	
1. 휴지통 이동 안내 및 실행
2. 목록에서 제외 처리

WALL-05	영구 삭제	휴지통에 있는 명함을 영구히 삭제한다.	
1. 휴지통 목록 UI
2. 영구 삭제 경고 모달

WALL-06	만난 장소 조회	지도에서 명함을 교환한 위치를 확인한다.	
1. 지도 SDK (Naver) 연동
2. 좌표에 마커 표시

CHAT-01	채팅방 조회	현재 진행 중인 채팅방 목록을 조회한다.	
1. 채팅방 리스트 UI
2. 안 읽은 메시지 표시

CHAT-02	채팅방 생성	초대할 사용자를 선택하여 채팅방을 만든다.	
1. 친구 선택 UI (Multi-select)
2. 방 생성 후 채팅 화면 이동

CHAT-03	채팅방 초대	현재 채팅방에 다른 사용자를 추가한다.	
1. 초대 가능 목록 모달
2. 초대 시 시스템 메시지 출력

CHAT-04	채팅방 나가기	현재 채팅방에서 나간다.	
1. 나가기 버튼 및 확인
2. 채팅 목록 갱신

NOTI-01	알림 조회	알림 목록을 확인한다.	
1. 알림 아이콘 배지 표시
2. 알림 리스트 UI

NOTI-02	알림 상세	알림 클릭 시 해당 페이지로 이동한다.	
1. 알림 타입별 라우팅 처리(채팅방, 명함상세 등)

NOTI-03	알림 관리	알림을 읽음 처리하거나 삭제한다.	
1. 모두 읽음/삭제 버튼 액션
2. UI 즉시 반영

PASS-01	임시 출입증 신청	관리자에게 임시 출입증 발급을 신청한다.	
1. 신청 사유/기간 입력 폼
2. 신청 완료 시 대기 상태 표시

PASS-02	출입증 사용	게이트에 폰을 태그하여 출입한다.	
1. NFC HCE 모드 활성화(출입 Key 송출)
2. 태깅 이펙트 진동으로 추가
3. 출입증 유효기간 카운트다운

ADM-PASS-01	임시 출입증 목록 조회	관리자는 사용자에게 발급된 임시 출입증 목록을 조회한다.	
1. 출입증 목록 테이블(Grid) 구현
2. 필터(유효/만료) 및 검색 UI

ADM-PASS-02	임시 출입증 발급	특정 사용자에게 유효기간과 권한을 설정하여 출입증을 발급한다.	
1. 사용자 검색 모달
2. 유효기간(Start~End) 설정
3. 출입 가능 게이트 선택 폼

ADM-PASS-03	출입 권한 부여	이미 발급된 출입증에 특정 NFC 기기(게이트) 접근 권한을 추가한다.	
1. 출입증 상세 모달 내 '게이트 추가'
2. NFC 기기 목록 체크박스 UI

ADM-PASS-04	출입 권한 회수	출입증에서 특정 NFC 기기의 접근 권한만 제거한다.	
1. 발급된 게이트 목록에서 '삭제(X)' 클릭
2. 변경 사항 저장 액션

ADM-PASS-05	임시 출입증 회수	발급된 임시 출입증을 만료 처리하여 즉시 사용할 수 없게 만든다.	
1. 목록에서 '회수' 버튼 클릭
2. 상태 'REVOKED'로 UI 갱신

ADM-SEC-01	보안팀 배정	NFC 기기마다 비인가 접근 시 호출될 보안팀 연락처를 등록한다.	
1. 기기 관리 페이지 내 '보안 담당자' 설정
2. 보안팀 연락처 입력/검색

SYS-01	 필수 권한 요청	앱 최초 실행 시 카메라, 위치, 앨범, 알림 접근 권한을 요청한다.	
1. OS 버전 체크
2. 권한 라이브러리 호출
 - POST_NOTIFICATIONS (알림)
 - READ_MEDIA_IMAGES (갤러리)
 - CAMERA (명함 촬영)
 - ACCESS_FINE_LOCATION (위치)

SYS-02	 권한 요청 거부 시 대응	권한 거부 시 앱이 종료된다. ㅎㅎ	System.exit(0) (강제 종료)

SYS-03	 NFC 활성 상태 체크	앱 최초 실행 시 NFC 활성 상태를 체크한다.	
1. NFC 활성화 상태 (NfcManager.isEnabled()) 체크
2. 비활성화 상태라면 알림 모달 출력
