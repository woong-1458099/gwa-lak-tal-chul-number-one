## 지라 사용법

1단계: 출근/작업 시작 (To Do ➔ In Progress)"저 이제 이 일 시작합니다! 건드리지 마세요!"라고 선언하는 단계입니다.티켓 가져오기: To Do(할 일)이나 Backlog에 있는 티켓 중 오늘 할 일을 고릅니다.담당자 할당(Assignee): 담당자를 **'나(Me)'**로 설정합니다. (누가 하는지 모르면 안 되니까요!)상태 변경: 티켓을 드래그해서 **In Progress(진행 중)**으로 옮깁니다.이걸 안 옮기고 일하면, 다른 팀원이 "어? 이거 아무도 안 하네? 내가 해야지" 하고 중복 작업을 하게 됩니다.

2단계: 작업 중 (In Progress)실제 코딩을 하는 단계입니다.스토리 포인트(Story Point) 입력: 만약 팀에서 일정 산정을 한다면, 이 티켓이 얼마나 걸릴지 점수(1점, 3점, 5점 등)를 입력합니다.이슈 발생 시: 작업하다가 기획서가 이상하거나 디자인 파일이 없으면, **댓글(Comment)**로 팀원(@멘션)을 호출해서 물어봅니다. 지라에 기록을 남겨야 나중에 딴소리가 안 나옵니다.

3단계: 작업 완료 & 검사 요청 (In Progress ➔ Review)코딩을 다 했고, 팀원들에게 검사(Merge)를 부탁하는 단계입니다.PR(MR) 생성: 깃랩/깃허브에 코드를 올리고 Merge Request 주소를 복사합니다.상태 변경: 티켓을 Code Review(코드 리뷰) 또는 Review 칸으로 옮깁니다.★댓글 남기기 (가장 중요):Git 자동 연동이 안 되어 있다면, PR 주소 링크를 댓글로 남겨주세요.예시: 작업 완료했습니다.

4단계: 진짜 끝 (Review ➔ Done)코드가 합쳐졌고(Merged), 기능이 잘 돌아가는지 확인까지 끝난 단계입니다.상태 변경: 티켓을 **Done(완료)**으로 옮깁니다.마음의 평화: Done 칸에 쌓인 티켓들을 보며 뿌듯함을 느낍니다. 


지라 사용 예시 (모범 답안)상황: 작성자님이 기본 틀 구성 티켓을 맡아서 처리할 때순서행동지라 상태(Status)1"나 이거 할게!" 하고 내 얼굴 박고 오른쪽으로 옮김In Progress2열심히 코드 짜고 Git에 Push 함In Progress3PR 올리고, 지라 티켓에 "PR 링크" 댓글로 남김In Progress4상태를 리뷰 단계로 옮기고 팀원한테 "리뷰해줘" 말함In Progress ➔ Review5(팀원이 Merge 해줌)Review6최종 확인 후 완료 처리Review ➔ Done
