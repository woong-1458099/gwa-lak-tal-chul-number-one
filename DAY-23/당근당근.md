문제 제목 : 22399 쉬운 당근 포장 ( 정말 쉬울까 ?)

문제 설명 : 
1. 당근은 크기를 기준으로만 구분한다. (크기 값 Ci)

2. 같은 크기의 당근은 반드시 같은 상자에만 들어가야 한다.
ex) 크기 1인 당근이 5개 있으면, 이 5개는 전부 소 or 전부 중 or 전부 대 중 한 상자에만 들어가야 함.

3. 비어 있는 상자가 있으면 안 된다.

4. 소/중/대 세 상자 모두 최소 1개 이상의 당근이 있어야 함.

5. 위 조건을 만족하면서,
세 상자에 들어 있는 당근 개수의 차이(최댓값 - 최솟값) 가 최소가 되도록 포장해야 한다.

만약 위 조건대로 포장하는 것이 불가능하면 -1을 출력한다.

문제 접근 방식: 당근을 크기 순서대로 정리해서 3개의 상자에 나누되, 같은 크기의 당근은 같은 상자에만 들어가고, 비어 있는 상자는 없으며,
세 상자의 개수 차이가 최소가 되게 만들자.”

필요한 알고리즘 개념 : 정렬, 완전탐색, 시간복잡도 이해 정도?


풀이과정
1. N 입력하고, 당근 크기 리스트 입력 해서 정렬시키고
2. 답 을 무한대로 초기화 하고, 포장 가능 여부를 possible = False 로 초기화
3. 첫 번째 구간 끝 i를 0~N-3 까지 돌린다
-> 만약 carrots[i] == carrots[i+1] 이라면 같은 크기를 두 구간에 나누게 되므로 continue
4. 두 번째 구간 끝 j를 i+1 ~ N-2 까지 돌린다.
-> 만약 carrots[j] == carrots[j+1] 라면 continue
5. 소.중.대 상자의 개수
small = i+1
medium = j-i
large = N-1-j
6. 세 상자 중 하나라도 비어있으면 continue (비어있으면 0)
